# Product & Technical Backlog

This backlog translates the HYROX Decision Alignment Platform roadmap into concrete, prioritized work items.

The backlog is organized by epics and vertical slices.

---

## Epic 1 — Athlete Foundation

Goal: Create athlete identity and context.

### Features

- Create athlete profile
- Update athlete profile
- View athlete profile

### Acceptance Criteria

- Athlete data is persisted
- Profile reflects category and experience
- Profile is editable

---

## Epic 2 — Competition Context

Goal: Anchor training decisions to a real event.

### Features

- Register competition
- Edit competition
- Select active competition

### Acceptance Criteria

- Only one active competition at a time
- Competition date influences planning

---

## Epic 3 — Weekly Volume Target

Goal: Define training expectations.

### Features

- Define weekly volume target
- Update weekly volume target
- View weekly volume target

### Acceptance Criteria

- Targets are week-based
- Targets include running, strength, mixed and hours

---

## Epic 4 — Training Session Logging

Goal: Capture training reality.

### Features

- Log training session
- Edit training session
- Delete training session
- Add exercises to session

### Acceptance Criteria

- Session impacts weekly volume
- Session stores intensity and effort

---

## Epic 5 — Volume Deduction Engine

Goal: Track remaining training capacity.

### Features

- Calculate volume contribution
- Deduct weekly volume
- Update weekly volume balance
- Display weekly volume status

### Acceptance Criteria

- Balance updates in real time
- Status reflects OnTrack / UnderTarget / OverTarget

---

## Epic 6 — Alignment Engine

Goal: Evaluate training vs competition.

### Features

- Evaluate weekly alignment
- Generate alignment score
- Generate strengths, gaps and risks

### Acceptance Criteria

- Alignment is explainable
- Score is consistent

---

## Epic 7 — Recommendation Engine

Goal: Provide actionable guidance.

### Features

- Generate recommendations
- Prioritize recommendations
- Generate explanations

### Acceptance Criteria

- Every recommendation has explanation
- Recommendations consider time constraints

---

## Epic 8 — Weekly Plan

Goal: Convert guidance into action.

### Features

- Generate weekly plan
- Display focus areas
- Suggest sessions

### Acceptance Criteria

- Plan reflects alignment gaps
- Plan respects remaining volume

---

## Epic 9 — AI Support Layer

Goal: Enhance intelligence safely.

### Features

- Pattern detection
- Natural language explanations
- Training trend analysis

### Acceptance Criteria

- AI never overrides business rules
- AI output is explainable

---

## Epic 10 — Education Layer

Goal: Teach athlete.

### Features

- Show decision explanations
- Show consequence scenarios
- Show weekly learning summary

---

## Epic 11 — External Integration

Goal: Reduce manual input.

### Features

- Garmin session import
- Data normalization
- Session reconciliation

---

## Epic 12 — Observability

Goal: System trust and stability.

### Features

- Event logging
- Error tracking
- Alignment audit trail

---

## Epic 13 — Security & Access

Goal: Protect athlete data.

### Features

- Authentication
- Authorization
- Data isolation

---

## Prioritization Strategy

Order of delivery:

1. Athlete Foundation
2. Competition Context
3. Weekly Volume Target
4. Training Session Logging
5. Volume Deduction Engine
6. Alignment Engine
7. Recommendation Engine
8. Weekly Plan
9. AI Support
10. Integrations

---

## Backlog Philosophy

- Vertical slices over technical layers
- Domain value first
- Education over automation
- Clarity over complexity

---

## Managerial View

This backlog ensures:

- Predictable delivery
- Domain validation
- Product learning
- Engineering discipline

---

## Final Statement

This backlog exists to transform training confusion into training clarity.
